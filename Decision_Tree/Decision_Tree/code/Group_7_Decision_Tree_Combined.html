The histograms of the diabetes dataset display the distributions of important health-related variables, revealing patterns such as skewness, unusual values, and potential predictors of diabetes outcomes. For instance:

Age: The Age variable shows a slight right skew, with most individuals between 20 and 40 years old. This indicates the dataset mainly includes young to middle-aged women. BMI: BMI shows a near-normal shape, with most individuals having BMI values between 25 and 40. This suggests that many participants fall in the overweight or obese category, which is strongly linked to diabetes risk. Because the distribution is fairly balanced, BMI is expected to be an important and stable predictor.

Glucose: Glucose appears as a smooth, approximately normal distribution, centered roughly around 100–140 mg/dL. After removing zeros, this variable becomes much cleaner and more interpretable. Glucose will likely be one of the strongest predictors of diabetes due to its clear and meaningful variation.

Outcome: The Outcome variable is binary, with more individuals classified as non-diabetic (0) than diabetic (1). This imbalance means a predictive model may need balancing techniques (oversampling, class weights) to avoid bias toward the majority class.

Pregnancies: pregnancies shows that it is Right-skewed, most individuals have 0–5 pregnancies.

Train-Test Split

The classification dataset is split similarly to ensure fair evaluation. This mirrors the regression approach but targets class labels.

set.seed(2025)
split_idx_clf <- caret::createDataPartition(diabetis_cln$Outcome, p = 0.75, list = FALSE)
train_clf <- diabetis_cln[split_idx_clf, ]
test_clf <- diabetis_cln[-split_idx_clf, ]

Build Classification Decision Tree

attach(diabetis_cln)
diabet_clf_tree <- rpart(
  Outcome~ .,
  data = train_clf,
  method = "class",
  control = rpart.control(minsplit = 20,   # need >=30 obs to try splitting
                          maxdepth = 8,    # limit tree depth to keep it readable
                          cp = 0.02)
)

A classification tree (method = “class”) is built, using metrics like Gini impurity to split data into classes. Features like rm and lstat again prove influential.

Visualize the Classification Tree

rpart.plot(
  diabet_clf_tree,
  type =2,
  extra = 104,
  fallen.leaves = TRUE,
  box.palette = "YlGnBl",
  shadow.col = "gray",
  cex = 0.8,
  main = "Initial Classification Tree"
)

Explanation

The classification tree shows that glucose level is the strongest factor in predicting whether a person has diabetes. The model first splits at Glucose < 128, because this threshold best separates people who are likely not diabetic from those who are likely diabetic. For people with low glucose, the tree then uses insulin and age to refine the prediction, while for those with high glucose, age appears again because it helps separate younger individuals with lower risk from older individuals with higher risk. Age appears multiple times in the tree because each split focuses on a different subgroup, and in each subgroup age happens to be the best variable for further separating the outcomes. Overall, the tree shows that glucose is the main indicator of diabetes, while insulin, age, and genetic factors help improve accuracy for more specific cases.

Prune the Tree to Avoid Overfitting

Pruning a decision tree to avoid overfitting means simplifying a complex, fully grown tree by removing branches (nodes) that capture noise, not patterns, making the model generalize better to new data.

# Find best cp and prune
best_cp_clf <- diabet_clf_tree$cptable[which.min(diabet_clf_tree$cptable[, "xerror"]), "CP"]
diabet_clf_tree_prune <- rpart::prune(diabet_clf_tree, cp = best_cp_clf)

# Check complexity parameters

Explanation

The complexity parameter (CP)

The Complexity Parameter (CP) in machine learning, especially for decision trees like CART, controls tree size by penalizing complexity; a larger CP leads to a smaller, simpler tree (pruned), while a smaller CP allows for a larger, more detailed tree, preventing overfitting by requiring significant improvement (relative error reduction) for new splits, often tuned via cross-validation to find the best balance for model performance.

Pruned Classification Tree

The pruned classification tree shows that glucose level is the dominant variable for predicting diabetes. With a single split at Glucose < 128, individuals with lower glucose levels are mostly classified as non-diabetic, while those with higher glucose levels are classified as diabetic. Pruning removes smaller, less meaningful splits and produces a simpler, more generalizable model that focuses on the most influential predictor.

Classification Report

Precision, recall, and F1-scores are tabulated. These metrics highlight class-specific performance, crucial for imbalanced data.
